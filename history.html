以前

使⽤Hugo+Gitbook+Nginx 构建静态博客⽹站

前⾔：之前我⼀直使⽤的是wordpress搭建的博客，由于是在某云搞活动时买的最低配置服务器，⽽wordpress⼜需要安 装很多组件，⽹站变得越来越慢。最近发现了Hugo 是 Go 编写的静态⽹站⽣成器，速度很快，依赖于 Markdown ⽂ 件，⾮常适合博客。

⼀、部署hugo

在Github上下载Hugo的Release包上传到Linux服务器

$ yum -y install git golang 
$ mkdir -p /app/hugo_0.74.3 
$ tar xf hugo_0.74.3_Linux-64bit.tar.gz -C /app/hugo_0.74.3 
$ cp /app/hugo_0.74.3/hugo /usr/local/bin/ 
$ hugo version Hugo Static Site Generator v0.74.3-DA0437B4 linux/amd64 BuildDate: 2020-07-23T16:22:34Z
创建⽹站
$ hugo new site zhanminblog[名称⾃定义] 
$ ls 
archetypes config.toml  content  data  layouts  static  themes

选择主题

在这⾥hugo themes选择你喜欢的主题并根据主题的REAMDE进⾏相应操作，⽐如我选择的是Pure主题

因为每个主题的样式和功能不同，所以要根据主题的README介绍进⾏调试，将主题的配置和⽂章⽬录复制到⽹站根⽬录下

$ rm -f /app/zhanminlog/config.toml 
$ cp -r themes/pure/exampleSite/* /app/zhanminblog/

根据⾃⼰的想法修改模板的配置⽂件进⾏调试，可参考主题的README⽂件，我的部分定义如下：

$ cat config.
baseURL: http://www.unmin.club 
theme: pure 
title: 前⾏ 
defaultContentLanguage: zh # en/zh/... 
footnoteReturnLinkContents: ↩ 
hasCJKLanguage: true 
paginate: 7 
enableEmoji: true 
PygmentsCodeFences: false 
googleAnalytics: "" # UA-XXXXXXXX-X 
permalinks:
    posts: /:year/:month/:filename/ 
taxonomies:
------------------------

创建第⼀篇⽂章

根据主题的定义，⽂章需要放在posts⽬录下，由于上⾯我们已经copy了主题模板的⽂件。所以我们可以直接新建⽂ 章，⽆需使⽤ hugo new posts/xxx.md 指令。

$ cd /app/zhanminblog/content/posts 
$ touch heelo.md 
$ cat hello.md

启动服务预览

由于我使⽤的是Linux公有云服务器，需要开启外机访问，如果是windows本机调试，则直接使⽤ 访问4000端⼝即可

$ hugo server --bind="0.0.0.0" -v -w -b http://www.unmin.club

当然这种启动⽅式也可直接使⽤到线上，直接修改或者创建md⽂件即可，⽆需使⽤build指令转化为html⽂件。

如果你是本地调试则可以使⽤ hugo 放到Github或者Nginx上即可访问。

命令将⽂件转化为html格式，会⽣成名为“public”的⽬录，然后将这个⽬录的内容

由于我还需要使⽤Gitbook来写⽂章，⽽且打算将这两个集成到⼀块，所以我选择的⽅式是直接在Linux上修改⽂章，通 过启动其他端⼝号来预览⽂章，然后在通过 hugo 转换为html，复制到Nginx的html⽬录下发布，最后通过nginx的80 端⼝访问。

指定端⼝号来预览⽂章

$ hugo server --bind="0.0.0.0" -v -w -p 8080 -b http://www.unmin.club

发布⽂章 预览没有问题则构建为html⽂件，⽣成public⽬录，注意使⽤

hugo

指令需要在⽹站的根⽬录。

$ hugo 
$ ls public/ 2020 about 404.html about-hugo about-us avatar.png categories contact css donate elk-book favicon.ico fonts index.html index.xml js

$ tar xf node-v12.18.1-linux-x64.tar.xz

$ export PATH=$PATH:/app/node-v12.18.1-linux-x64/bin

配置Nginx

ok，现在我们修改Nginx的配置⽂件，添加gitbook的⽬录地址

$ vim /etc/nginx/nginx.conf 
--------------------
        server { 
                listen 80 default_server; 
                listen [::]:80 default_server; 
                server_name _; 
                root /usr/share/nginx/html;
                # Load configuration files for the default server block. 
                include /etc/nginx/default.d/*.conf;

            location / { 
            }

                error_page 404 /404.html; 
                        location = /40x.html { 
                }

                error_page 500 502 503 504 /50x.html; 
                        location = /50x.html { 
                }

                location /k8s-book { ## 新增加gitbook访问路径 
                alias /app/k8s-gitbook/_book; # 这个路径是关键，就是GitBook⼯程下build出来的_book⽬录，需要运⾏gitbook build命令⾃动⽣成 
                index index.html index.htm; 
                autoindex on; 
--------------------
